# betaDispersionInterpretability

R package with the code and analyses for the paper *"Beta dispersion lacks interpretability for differential stochastic dispersion analyses."* Using a generative compositional beta-binomial model fit via `sccomp`, we benchmark PERMDISP (`vegan::betadisper`) across the standard distance choices (Bray–Curtis, Aitchison, robust Aitchison, Jaccard, Hellinger) under controlled differential-abundance and differential-dispersion scenarios.

## Layout

```
.
├── DESCRIPTION                          # package metadata + dependencies
├── NAMESPACE                            # generated by roxygen2
├── R/                                   # exported helpers (28 functions)
│   ├── simulation.R                     # simulate_compositional_bb + helpers
│   ├── sccomp.R                         # sccomp fit -> simulator inputs
│   ├── pipeline.R                       # per-job runners + FPR / sweep summaries
│   ├── betadisper.R                     # distance + permutest wrappers
│   ├── permanova.R                      # adonis2 wrappers + TPR summary
│   ├── alpha_diversity.R                # richness / Shannon / Pielou + plot
│   ├── design.R, transforms.R,
│   └── plot_labels.R                    # small utilities
├── tests/testthat/                      # strict snapshot tests for R/
└── vignettes/<cohort>/                  # per-cohort benchmark reports
    ├── BritoIL_2016/                    #   benchmark_paper_analyses_<cohort>.qmd
    ├── HMP_2019/                        #   + a _targets store after first render
    ├── ShaoY_2019/
    └── ZellerG_2014/
```

## Install

```r
options(repos = BiocManager::repositories())
devtools::install_github("MangiolaLaboratory/beta_dispersion_interpretability")
```

The Bioconductor `repos` setting is needed because some `Imports:` (`mia`, `S4Vectors`, `SummarizedExperiment`, `sccomp`) live on Bioconductor. For development from a clone, `devtools::install(".")` or `devtools::load_all(".")`.

## Reproducing a cohort report

```bash
quarto render vignettes/HMP_2019/benchmark_paper_analyses_HMP_2019.qmd \
  --execute-dir vignettes/HMP_2019
```

Each report calls `targets::tar_make()` on its own pipeline and writes a store next to the `.qmd`. Inputs (sccomp result `.rds` and cohort metadata) are not committed; obtain them per the methods section. A full `HMP_2019` render is the heaviest (~40 GB RAM, hours).

## Tests

```bash
Rscript -e 'devtools::test()'
```

Snapshot tests pin exact numerical outputs of every helper in `R/` so the simulator and analysis pipeline can be safely refactored.
